VASTQ03

Recording performance benchmarking tool

Introduction

Customers are often unsure about how to select the right storage device for their surveillance system. In addition, they may not know what is the maximum number of channels that can be managed without affecting recording performance and having to worry about dropped frames due to a storage device's insufficient write speeds. To assist customers in resolving these uncertainties, VIVOTEK provides a software tool called VVTKRecordingBenchmark.exe to test the system and evaluate its recording performance.

How to use VVTKRecordingBenchmark?

The VVTKRecordingBenchmark program is a software tool that uses a 3gp file generated by the ST7501 recording server to test recording performance. The program will automatically scan the 3gp file's properties and use the appropriate write speed to test recording performance. Refer to the following steps to use the program.

  1. Run ST7501, insert ONE camera, and record a few files.
  2. Stop ST7501 by navigating to Windows' Services. Locate VAST Uranus Watch Dog (or ST7501 Uranus Watch Dog) and click on Stop the service.
  3. Download the VVTKRecordingBenchmark program and unzip it.
  4. Place the VVTKRecordingBenchmark.exe file in the same folder in which you wish to save ST7501 recording files.
  5. Double-click the VVTKRecordingBenchmark.exe file to run it.
  6. Click the Browse button and select the folder where you wish to save ST7501 recording files.
  7. The program will automatically select a 3gp file, which is displayed in the Pattern field.
  8. Initial: This value represents the numbers of channels that the ST7501/VAST server is writing simultaneously to the HDD. For example, if the value of Initial is 16, this indicates that ST7501/VAST server is recording 16 channels of video from cameras to the HDD (storage).
  9. Loop: This value represents the number of testing runs that the program will perform to benchmark recording performance. For example, if Loop=2, and Initial=16, then it will perform two runs, with the testing sequence as follows:
    • First run: Test performance when recording 16 channels to HDD (storage).
    • Second run: Again test performance when recording 16 channels to HDD (storage).
    • The program will thus test recording of 16 channels to the HDD(storage) twice. More reliable results can be obtained if identical tests are repeated in this manner.
  10. Step: This feature is used in conjunction with Loop, and indicates the number of channels that will be added to the number specified in Loop for each testing run. For example, if the settings used are Initial=16, Loop=3, and Step=5, the test sequence will be:
    • First run: Test performance when recording 16 channels to HDD (storage).
    • Second run: Test performance when recording 21 (16+5) channels to HDD (storage).
    • Third run: Test performance when recording 26 channels (21+5) to HDD (storage).
    • By using this feature, you can test the maximum number of channels your PC is capable of simultaneously recording to the HDD (storage).
  11. Before clicking the Run button, remember to close the folder in which the recording files are saved.

    Notes

    Before using the program, be aware of the following points:

    1. Test results are FOR REFERENCE ONLY. The system's recording performance will vary depending on the following factors:
      1. Whether and what particular other applications are running on the system (PC).
      2. When the HDD (storage) is nearly full, write speed may decline (to a degree dependent on the manufacturer and model) and thereby affect recording performance.
      3. If Playback is used to view the video as it is recorded, recording performance may also be affected since data is being written to and read from the HDD (storage) simultaneously.
    2. Test results assume that all channels are using the same model of camera with identical video settings. Therefore, it is recommended that a single model be chosen from among the cameras in use at your site-and moreover that it be the one with the highest bit rate-as the baseline for benchmarking purposes. For example, if your surveillance system includes IP8161, IP8332, FD8134 models, and the IP8161 has the highest bit rate, then the IP8161 is the most appropriate choice as the baseline camera.
    3. Following up on note 2, the particular 3gp file used for benchmarking is important. We recommend that a file with video filmed by the baseline camera at night be used. At night, due to the low-light conditions, signal noise will be present, producing a greater quantity of data than in video filmed during the daytime, thereby providing a more conservative benchmarking result.
    4. The higher the values set for the baseline camera's video size and bit rate, the shorter the run time of the benchmark test will be, because the program uses the write speed of the 3gp file to measure recording performance.
    5. If in actual use, you manage 16 channels using ST7501/VAST, then we recommending using a value of 32 for Initial when testing recording performance.
