How to use Anystream

Introduction

VIVOTEK cameras offer flexible streaming options. User can dynamically select the desired codec, frame rate, resolution, and quality. Moreover, it is not necessary to go to a camera's configuration page and modify settings to do so. This document describes how to access and use these streaming options.

  1. How to check if a camera supports anystream.
    1. Using a CGI command
      1. Open your web browser.
      2. Enter this URL in the address field: http://<IP Address>/cgi-bin/admin/getparam.cgi?capability_nanystream
      3. If the value of the parameter "capability_nanystream" is shown as '1', the camera supports anystream.
    2. Using the camera's interface page
      1. Navigate to Configuration > Network > Streaming protocols
      2. If "videoany.mjpg" or "liveany.sdp" is shown as an access name, the camera supports anystream.
  2. How to use anystream. (The following uses the procedure for the IP8161 as an example.)
    1. Access name for anystream
      1. RTSP Streaming – The access names for stream 1, stream 2, stream 3, stream 4, and stream 5 (anystream) are live.sdp, live2.sdp, live3.sdp, live4.sdp and liveany.sdp, respectively.
      2. HTTP Server Push – The access names for stream 1, stream 2, stream 3, stream 4, and stream 5 (anystream) are video.mjpg, video2.mjpg, video3.mpg, video4.mpg and videoany.mjpg, respectively.
    2. URL
      1. Parameters of all codecs –
        1. codectype: h264, mpeg4 or mjpeg
        2. resolution: 176x144, 320x240, 640x480, 800x600, 1280x960, 1280x1024 or 1600x1200
      2. Parameters of H.264 –
        1. h264_intraperiod: 250, 500, 1000, 2000, 3000 or 4000
        2. h264_ratecontrolmode: vbr or cbr
        3. h264_quant: 1, 2, 3, 4, 5 or 99
        4. h264_qvalue: 0 ~ 51 (Note: qvalue must be present if quant is equal to 99)
        5. h264_bitrate: 1000~8000000
        6. h264_maxframe: 1 ~ 30
      3. Parameters of MPEG-4 –
        1. mpeg4_intraperiod: 250, 500, 1000, 2000, 3000 or 4000
        2. mpeg4_ratecontrolmode: vbr or cbr
        3. mpeg4_quant: 1, 2, 3, 4, 5 or 99
        4. mpeg4_qvalue: 2 ~ 31 (Note: qvalue must be present if quant is equal to 99)
        5. mpeg4_bitrate: 4000~8000000
        6. mpeg4_maxframe: 1 ~ 30
      4. Parameters of Motion JPEG –
        1. mjpeg_quant: 1, 2, 3, 4, 5 or 99
        2. mjpeg_qvalue: 2 ~ 97
        3. mjpeg_maxframe: 1 ~ 30
      5. Examples –
        1. Accessing a MPEG-4 stream: rtsp://<IP Address>/liveany.sdp?codectype=mpeg4&mpeg4_intraperiod=2000&mpeg4_quant=99&mpeg4_qvalue=23
        2. Accessing a H.264 stream: rtsp://<IP Address>/liveany.sdp?codectype=h264&h264_intraperiod=4000&h264_quant=99&h264_qvalue=40
        3. Accessing a Motion JPEG stream: http://<IP Address>/videoany.mjpg?codectype=mjpeg&mjpeg_quant=99&mjpeg_qvalue=50
    3. Media player or web browser
      1. RTSP streaming – QuickTime, VLC player, or other RTSP players.
      2. HTTP server push - VLC player, Firefox, or other browser supporting server push.
